Q.13 What are the ways to integrate CSS as a web page?

1. Inline-Style

The first way to incorporate CSS into HTML is using a technique known as inline-styling. When CSS rules are added directly into HTML elements (tags), this is referred to as inline styling.

For instance, suppose I want to alter an element's text colour:

<h1 style="color: red">Test Headline</h1>

The new rule (in this case, text colour) will then be applied to the h1 element:

Test Headline

2. Internal CSS

The internal CSS method is the second approach to include CSS in HTML. This method requires the usage of an HTML tag called "style" (not "style attribute"), between which we can place our CSS selectors and rules:

<style>
h1 {
color: red;
}
</style>
<body>
<h1>Test Headline</h1>
</body>

Test Headline

3. External CSS

The ideal practise is to keep HTML and CSS separate. In actual programming, HTML, CSS, and JavaScript must all be kept in distinct files and imported as needed. This method facilitates code maintenance and enhances readability.

To employ this method, we must first create individual CSS files with the.css extension and then connect them to HTML.

We could, for instance, make a CSS file called index.css. Our CSS rules are written in the index.css file:

h1 {
color: red;
}

Then we can import index.css to HTML with a <link> tag like below:

<head>
<link rel="stylesheet" type="text/css" href="index.css">
</head>
<body>
<h1> Test Headline </h1>
</body>

Q.17 What is the meaning of the CSS selector?

The collection of characters and special symbols known as CSS selectors tells the browser which CSS properties should be applied to which HTML element. CSS selectors employ the HTML element's attributes, class name, kind of tag (such as header, div, etc.), and id name (which starts with #) to choose which ones to select. The subject of the selector is the HTML element(s) that the selector has chosen.

To "find" (or choose) the HTML components you want to style, CSS selectors are employed.

CSS selectors can be categorised into one of five groups:

  • Select components using simple selectors (based on name, id, and class)
  • Combinator selectors (which choose elements based on how closely they are related to one another)
  • Elements can be chosen using pseudo-class selectors based on a specific state.
  • Selecting and styling pseudo-elements (a portion of an element)
  • Elements can be chosen based on an attribute or attribute value using attribute selectors.

Q.18 What are the media types allowed by CSS?

CSS3 Introduced Media Queries

CSS3's media queries expanded on the concept of media types introduced in CSS2 by looking at a device's capabilities rather than its type.

Media queries can be used to check many things, such as:

  • Viewport dimensions: width and height
  • Device dimensions: width and height
  • The tablet or phone's orientationâ€”is it in landscape or portrait mode? resolution

A common method for distributing a customised style sheet to desktops, laptops, tablets, and mobile phones (such as iPhone and Android phones) is to use media queries.

Media Query Syntax

A media type and one or more expressions that can resolve to true or false make up a media query.

@media not|only mediatype and (expressions) {
 CSS-Code;
}

If the media type supplied matches the type of device the document is being displayed on and all of the media query's expressions are true, the result of the query is true. A media query that returns true applies the associated style sheet or style rules in accordance with the standard cascade rules.

The media type is optional, and the all type will be presumed unless the not or only operators are used.

Additionally, you can use various stylesheets for various media:

<link rel="stylesheet" media="mediatype and|not|only (expressions)" href="print.css">

CSS3 Media Types

Value Description
all Used for all media type devices
print Used for printers
screen Used for computer screens, tablets, smart-phones etc.
speech Used for screenreaders that "reads" the page out loud

Media Queries Simple Examples

Including an alternative CSS section right inside your style sheet is one approach to leverage media queries.

The background colour in the example below is changed to lightgreen if the viewport is 480 pixels wide or more (pink if the viewport is less than 480 pixels wide):

@media screen and (min-width: 480px) {
    body {
       background-color: lightgreen;
   }
}
